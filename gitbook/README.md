# はじめに

## What is INTMAX?

ブロックチェーン上のプライバシーを実現するアプローチとして多くのプロジェクトはネットワーク上の 「 *データを暗号化して守る* 」方法を採用しています。ネットワーク上にState（ ≈ 状態）を保持しつつも、暗号技術でその中身を秘匿するというアプローチです。

**INTMAX** はそれらとは異なり ***ネットワーク上に State を保持しない*** 完全な Stateless Architecture で構築されています。  
データの正当性はZero-Knowledge Proof（ゼロ知識証明）によって検証され、ユーザー自身がプライバシーを秘匿したまま自分のデータ（ ≈ 資産）を管理できます。  

この Stateless Architecture がもたらすのは、**構造的なプライバシー**で、保護すべきデータがネットワーク上に存在しない以上、漏洩や検閲リスクそのものが排除されると同時にスケーラビリティも向上するという合理的な帰結（必然性）です。

### 特徴

- **構造的プライバシー** : プライバシーデータを隠すのではなく、そもそも存在させない
- **高いスケーラビリティ** : ノード負荷を最小化し、処理効率の大幅向上
- **低コストな送受信** : 低コスト + より速く・より安全な送受信
- **ユーザー主権の設計思想** : プロトコル（運営・中央管理者）側ではなく、主権はあくまでユーザー側
